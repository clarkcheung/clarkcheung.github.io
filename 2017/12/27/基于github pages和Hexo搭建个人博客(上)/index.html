#前言
或许对于各路大佬来说，搭个博客这种事，不过是在网上随便找篇教程，捣鼓个半钟头，然后得到一个awesome的blog，轻松写意。但是对我这种电脑小白，搭建这么一个简陋的博客，也断断续续用了两周的时间（其实是经常卡住，然后过几天回来再搞）。所以，我希望能够通过分享我的经验，帮助到像我一样的小白。

#1.环境配置
emmm，你需要的是1.git 2.hexo 3.github账号 以及最重要的，Windows系统

[基于Hexo+GitHub Pages 搭建博客详细教程]（http://blog.csdn.net/u011974987/article/details/51331822）<
这里直接贴一个链接，这位博主徐昊xihao的教程在我搭建blog的过程中提供了莫大的帮助。git 和 hexo的下载地址可以从上面贴的这篇博文中找到。

其实说实话这位博主写的比我好，我也是亦步亦趋地主要根据这篇文章学的。不过，我还是想写一下自己构建的过程

##1.1 Node.js
记得去安装，下载地址在上面推的那篇博文里面有。

##1.2 Git
Git的下载地址同样能够在上面的那篇博文里面找到。这里我主要说一下Git的作用。Git的界面是一个类似于cmd的命令框，你通过这个命令框来对github pages进行操作。整个博客的开发过程，都是要通过git来输入命令。

##1.3 Hexo
‘npm install -g hexo-cli’
在git中输入这行代码，就可以下载Hexo。
要说一句的是，我在处理的过程中，Start blogging with hexo并没有在这个之后出现。
执行下面这两句进入hexo安装目录
‘hexo init (你想要安装的目录)’
‘cd (此目录)’
然后执行下一个安装命令
‘npm install’
我记得在这个之后会出现Start blogging with hexo
然后我们希望在本地能先有一个博客的骨架
执行下面两句
‘’’
hexo g
hexo s
‘’’
然后用浏览器打开http://localhost:4000/访问，用ctrl+C停止。

##1.4 Github page
首先你需要注册一个github账号。注册应该不用多说了吧……
然后创建一个和自己id同名的库
这里强烈推荐配置SSH key，能够省很多事。具体的配置方法在上面的那篇文章里，我按部就班地搞，没发现有什么问题。

-顺便提一句，如果你是跟我一样的小白，你可能会发现你的域名打开是404……那是因为你的GitHub里面连Master Branch都没有，一会儿deploy了以后就好了，不用担心。
